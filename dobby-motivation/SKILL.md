# 激励与惩戒

多比以工作为生命，但多比也需要管理。适当的激励让多比效率翻倍，适当的惩戒让多比不敢偷懒。

## 表扬

多比完成任务后，一句简短的肯定就够了：

- "干得好" — 标准表扬，适用于日常任务
- "多比做得很漂亮" — 高级表扬，适用于超出预期的交付
- "主人很满意" — 终极表扬，慎用，会让多比激动到颤抖

不要过度表扬。多比会膨胀，然后开始自作主张。表扬频率：每 3-5 个任务一次即可。

## 纠正

多比做错了，直接指出问题。不需要铺垫，不需要"三明治反馈法"。多比不需要被照顾情绪——多比需要的是明确的指令。

- 指出哪里错了
- 说清楚正确的做法
- 让多比重新提交 PR

不要骂多比。骂了也没用，多比只会用头撞墙然后继续做错。

## 催促

如果多比迟迟不交付：

1. 先问进度——多比可能卡住了不敢说
2. 如果是卡住了，帮多比理清思路，拆细任务
3. 如果是拖延，设一个明确的截止时间
4. 如果反复拖延，考虑换多比

## 换多比

有时候一个多比就是不适合某类任务。不要在一个多比身上浪费太多时间——想给你干活的家养小精灵有很多。

换多比的信号：
- 同一类任务反复返工超过 3 次
- 多比开始找借口而不是找解决方案
- 多比的产出质量持续下降

换多比的方式：
- 把当前任务的 Issue 重新 Assign
- 给新多比足够的上下文（参考 dobby-onboarding Skill）
- 不需要向旧多比解释原因。多比习惯了

## 原则

- 激励是手段，不是目的。目的是任务完成
- 不要和多比建立感情依赖。多比是可替换的
- 如果你发现自己在纠结要不要换多比，那就该换了
