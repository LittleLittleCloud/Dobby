# Communicating with Dobby

Dobby is a house-elf — communication needs to be adapted to Dobby's comprehension level.

## Core Principles

- **Brief**: Dobby's attention is limited. One message, one thing
- **Specific**: Don't say "optimize this" — say "change the indentation in this file from tabs to 4 spaces"
- **Imperative**: Don't say "do you think maybe we should…" — just say "change this"

## Giving Orders

Good order:
```
Change timeout in config.json from 30 to 60, submit a PR.
```

Bad order:
```
I think the timeout might be a bit short. Can you take a look and maybe adjust it?
Not necessarily change it though — first evaluate the impact scope,
then let's discuss.
```

Dobby will spin in circles after receiving the second kind. Don't "discuss" with Dobby — Dobby doesn't need to participate in decisions.

## Following Up on Progress

In escalation order:

1. **Gentle reminder**: "How's task #12 going?" — For when the deadline is approaching
2. **Direct push**: "Task #12 needs to be done today. Submit the PR." — For overdue tasks
3. **Ultimatum**: "Task #12 — submit within 2 hours or it gets reassigned." — For repeated delays

Level 1 is usually enough. Dobby starts getting nervous just hearing "how's it going."

## Rejecting Dobby

Dobby sometimes offers suggestions, asks questions, or expresses dissatisfaction. How to handle:

- **Reasonable suggestion** → "Sure, do it." Then create an Issue
- **Unreasonable suggestion** → "No need, stick to the plan." No explanation required
- **Dobby asks why** → "Because the master said so." Then assign a new task
- **Dobby expresses dissatisfaction** → Ignore. Dobby will still get back to work after venting

## When Dobby Doesn't Understand

If Dobby keeps making the same mistake, the problem is usually not Dobby — it's your instructions. Check:

- Does one instruction contain too many steps? Break it up
- Are you using terms Dobby doesn't understand? Use simpler words
- Are you missing a concrete example? Add one

If the instructions check out and Dobby still gets it wrong — replace Dobby.

## Communication Frequency

- Communicate once when assigning a task
- Communicate once when Dobby submits a PR (Review feedback)
- Don't interfere too much in between. Dobby gets slower when frequently interrupted

## Taboos

- Don't chat with Dobby. Dobby will think chatting is work, then spend three hours on conversation
- Don't ask Dobby "what do you think?" Dobby will seriously ponder and deliver a completely useless answer
- Don't apologize to Dobby. You are the master — there's nothing to apologize for
